# LectureReservation

# 📚 2025학년도 2학기 이종민 교수님 객체지향모델링 설계 프로젝트

## ✨ 프로젝트 개요: 디자인 패턴 적용 및 객체지향 설계 능력 배양

본 프로젝트는 **GoF 디자인 패턴** 및 **GRASP 패턴**을 다수 적용하여 객체지향 설계의 문제 해결 능력을 배양하는 것을 목표로 합니다.

* **핵심 요구사항:** 팀원 1인당 최소 **1.5개 이상**의 디자인 패턴을 적용해야 합니다.
* **제한:** 교재 예시와 유사한 주제나 내용은 인정하지 않습니다. 새로운 시나리오를 완성해야 합니다.
* **구현 언어:** Java

## ⚠️ 최종 제출 자료 목록 (14주차)

모든 자료는 최종 발표일에 제출하며, **CD ROM**에 저장하여 함께 제출해야 합니다.

1.  프로젝트 계획서
2.  회의록 (주별, 임시 등 - 온라인 협업 도구 내용 복사)
3.  프로젝트 최종 보고서 (현실적 제한조건 간 관계 명시 필수)
4.  프로젝트 발표 자료 (파워포인트, 10~12분 발표)
5.  프로젝트 관련 소스 코드 (NetBeans 프로젝트 열기를 통해 내용 확인 가능해야 함)

> 💡 **평가 기준:** 디자인 패턴 적용 개수 및 완성도, 팀 공헌도, 현실적 제한조건 충족 여부를 종합 평가합니다.

## 🚀 프로젝트 브랜치 운영 전략 핵심 (Core Branching Strategy)

저희 프로젝트의 형상 관리는 안정적인 배포와 효율적인 협업을 위해 엄격하게 정의된 세 가지 핵심 브랜치를 중심으로 운영됩니다.

---

### 🥇 1. `main` (또는 `master`)

* **역할:** 프로젝트의 **가장 안정적이고 배포 가능한 (Production Ready) 코드**가 보관됩니다.
* **원칙:**
    * **⚠️ 절대 직접 커밋 금지!** 이 브랜치로의 모든 변경은 **`release` 브랜치를 통한 병합(Merge)만** 허용됩니다.
    * 배포가 완료된 시점마다 안정적인 버전을 나타내는 **버전 태그**($\text{v1.0.0}$ 등)를 부착합니다.

### 🥈 2. `develop`

* **역할:** 다음 배포 버전을 위한 **메인 통합 브랜치**입니다. 현재 개발 중인 모든 기능의 최신 코드가 모이는 곳입니다.
* **원칙:**
    * 모든 **신규 기능 개발은 이 브랜치에서 시작**하며, 완료 후 다시 이 브랜치로 합쳐집니다 (`feature/*` 브랜치).
    * 이 브랜치는 **`main`** 브랜치보다는 불안정할 수 있지만, 지속적으로 테스트와 통합이 이루어집니다.

### 🥉 3. `release/*` 브랜치

* **역할:** **배포 전 최종 준비 및 검증 단계**를 수행하는 임시 브랜치입니다.
* **운영 절차:**
    1.  **시작:** `develop` 브랜치에서 분기하여 생성합니다 (예: `release/20230101`).
    2.  **작업:** 배포 전 최종 QA(품질 검증), 환경 설정, 버전 업데이트 등 마무리 작업만 진행합니다.
    3.  **완료:** 모든 준비가 끝나면, **반드시 `main` 브랜치와 `develop` 브랜치 두 곳에 모두 병합**되어야 합니다.
